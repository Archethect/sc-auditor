# Codex CLI Setup

Guide for using sc-auditor v2.0.0 with [OpenAI Codex CLI](https://github.com/openai/codex).

> **REQUIRED: Multi-Agent Mode.** sc-auditor dispatches parallel agents during HUNT (5-6 lanes), ATTACK, and VERIFY phases. Without `multi_agent = true` in your Codex `config.toml`, all phases run sequentially and audits take 3-5x longer.

## Quick Start

1. **Install the MCP server:**

   ```bash
   codex mcp add sc-auditor -- npx -y sc-auditor
   ```

2. **Enable multi-agent mode** in your Codex `config.toml`:

   ```toml
   [agent]
   multi_agent = true
   ```

3. **Set your Solodit API key:**

   ```bash
   export SOLODIT_API_KEY="your-key-here"
   ```

4. **Launch an audit:**

   ```
   /security-auditor src/
   ```

## Prerequisites

- **Node.js >= 22**
- **Codex CLI** — See [Codex docs](https://github.com/openai/codex) for installation
- **Solodit API Key** — Required for `search_findings`. Get one at [solodit.cyfrin.io](https://solodit.cyfrin.io) > API Keys

### Optional

- **Slither** — `pip install slither-analyzer` (+ `solc-select` for compiler management)
- **Aderyn** — `cargo install aderyn`
- **Foundry** — For PoC generation ([getfoundry.sh](https://getfoundry.sh/))
- **Echidna** / **Medusa** / **Halmos** — Fuzz testing and symbolic execution

## Installation

### From npm (npx)

The Quick Start command above handles this. Alternatively, register manually:

```bash
codex mcp add sc-auditor -- npx -y sc-auditor
```

### From Source

```bash
git clone https://github.com/Archethect/sc-auditor.git
cd sc-auditor && npm install && npm run build
codex mcp add sc-auditor -- node /path/to/sc-auditor/dist/mcp/main.js
```

## Environment Variables

```bash
export SOLODIT_API_KEY="your-key-here"
```

Or create a `.env` file in the project you are auditing:

```
SOLODIT_API_KEY=your-key-here
```

## Configuration

sc-auditor reads optional settings from `config.json` in the project root. All fields have sensible defaults.

```json
{
  "default_severity": ["CRITICAL", "HIGH", "MEDIUM"],
  "default_quality_score": 2,
  "static_analysis": {
    "slither_enabled": true,
    "aderyn_enabled": true
  },
  "workflow": {
    "mode": "default",
    "parallel_hunters": true
  }
}
```

See the main [README](../README.md#configuration) for all configuration options.

## Skill Installation

The `.agents/` directory is auto-generated by `npm run build` from the `skills/` source of truth.

### From npm (npx)

The `.agents/` directory is included in the published package. Copy it to your project:

```bash
cp -r node_modules/sc-auditor/.agents/ .agents/
```

### From Source

```bash
git clone https://github.com/Archethect/sc-auditor.git
cd sc-auditor && npm install && npm run build
cp -r .agents/ /path/to/your/project/.agents/
```

### Usage

1. Copy `.agents/skills/security-auditor/` to your project root or `~/.agents/skills/`
2. Invoke: `/security-auditor src/contracts/`

## Tool Timeout

Static analysis tools can take several minutes on large projects. Increase the timeout in your Codex `config.toml`:

```toml
[mcp.sc-auditor]
tool_timeout_sec = 300
```

| Tool | Description |
|------|-------------|
| `run-slither` | Slither static analysis |
| `run-aderyn` | Aderyn static analysis |
| `get_checklist` | Cyfrin audit checklist (optional category filter) |

| `generate-foundry-poc` | Foundry PoC scaffold generation |

| `run-medusa` | Medusa fuzz testing |
| `run-halmos` | Halmos symbolic execution |

## Troubleshooting

### Tools not found

Verify the MCP server is registered: `codex mcp list` should show `sc-auditor`.

### Phases running sequentially

Enable `multi_agent = true` in your Codex config.toml. Without this, HUNT, ATTACK, and VERIFY phases cannot dispatch parallel agents.

### Solodit API key errors

Set `SOLODIT_API_KEY` in your environment or `.env` file.

### Slither/Aderyn timeouts

Increase `tool_timeout_sec` in your Codex config.toml (see above).

### Environment variable forwarding

Codex may not forward all env vars to MCP servers. If `SOLODIT_API_KEY` is not picked up, use a `.env` file in the project root.

## Verification Checklist

After running an audit with Codex, verify:
- [ ] Phases ran in order (RESOLVE → SETUP → MAP → HUNT → ATTACK → VERIFY → REPORT)
- [ ] Orchestrator stopped after MAP for your review
- [ ] Orchestrator stopped after HUNT for hotspot selection
- [ ] Sub-agents returned JSON (not prose)
- [ ] Checkpoint files exist in `.sc-auditor-work/checkpoints/`
- [ ] Findings include `da_attack` and `da_verify` fields
